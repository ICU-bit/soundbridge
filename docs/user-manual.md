## 1. 快速开始

### 1.1 系统要求

| 平台 | 最低版本 | 推荐版本 |
|------|----------|----------|
| Windows | Windows 10 1809 | Windows 11 |
| Android | Android 8.0 (API 26) | Android 12+ |

**硬件要求：**
- WiFi 802.11n 或更高（推荐 5GHz 频段）
- 蓝牙 4.0+（BLE）或蓝牙 2.1+（经典蓝牙）
- 两端设备需在同一局域网，或通过热点/USB/蓝牙直连

### 1.2 下载安装

**Windows 端：**

1. 从 Release 页面下载 `SoundBridge-Setup-x64.msi`
2. 双击运行安装程序，按提示完成安装
3. 安装完成后，SoundBridge 自动添加到系统托盘
4. （可选）在设置中开启"开机自启动"

**Android 端：**

1. 从 Release 页面下载 `SoundBridge-x.x.x.apk`
2. 在手机上打开 APK 文件进行安装
3. 如提示"未知来源"，请在系统设置中允许安装
4. 首次启动需授予麦克风和网络权限

### 1.3 首次连接

最简步骤（WiFi 局域网模式）：

1. **确保两端设备连接到同一个 WiFi 网络**
2. **Android 端**：打开 SoundBridge App，点击"开始传输"
3. **Windows 端**：右键系统托盘图标，查看"设备管理"
4. 等待自动发现 Android 设备（通常 2-5 秒）
5. 点击设备名称进行连接
6. 连接成功后，托盘图标变为绿色，状态显示"已连接"

## 2. 连接方式

SoundBridge 支持四种连接方式，按推荐程度排序：

### 2.1 WiFi 局域网（推荐）

**适用场景：** 家庭或办公室，两端在同一 WiFi 网络下。

**特点：**
- 自动发现设备（mDNS），无需手动输入 IP
- 延迟最低（约 30-50ms）
- 支持高音质模式

**操作步骤：**
1. 确保 Windows 和 Android 连接到同一 WiFi
2. Android 端打开 SoundBridge，点击"开始传输"
3. Windows 端托盘菜单 → 设备管理 → 等待自动发现
4. 点击设备名称连接

**如果自动发现失败：**
- 检查路由器是否开启 AP 隔离（需关闭）
- 手动添加设备：输入 Android 端显示的 IP 地址

### 2.2 WiFi 热点

**适用场景：** 没有路由器，手机直接开热点给电脑连。

**特点：**
- 无需外部网络设备
- 手机作为热点，电脑连接后自动发现

**操作步骤：**
1. **Android 端**：设置 → 连接方式 → 选择"WiFi 热点"
2. 点击"创建热点"，设置热点名称和密码
3. **Windows 端**：连接到手机创建的 WiFi 热点
4. Windows 端打开 SoundBridge，自动发现并连接

**注意事项：**
- 热点模式下手机无法同时使用移动数据上网
- 建议使用 5GHz 频段以获得更低延迟

### 2.3 USB/ADB

**适用场景：** 需要最稳定连接，或 WiFi 环境不可用。

**特点：**
- 零丢包，延迟稳定
- 需要 USB 数据线
- 需开启 USB 调试

**操作步骤：**
1. **Android 端**：设置 → 开发者选项 → 开启"USB 调试"
2. 用 USB 数据线连接手机和电脑
3. 手机弹出"允许 USB 调试"提示，点击"允许"
4. **Windows 端**：设置 → 连接方式 → 选择"USB/ADB"
5. 点击"ADB 连接"，等待端口转发建立
6. 连接成功后即可开始音频传输

**前提条件：**
- 已安装 ADB 驱动（Windows 通常自动安装）
- 手机已开启开发者选项和 USB 调试

### 2.4 蓝牙

**适用场景：** 无 WiFi 环境，短距离传输。

**特点：**
- 无需 WiFi 网络
- 延迟略高于 WiFi（约 50-80ms）
- 传输距离约 10 米

**操作步骤：**
1. **Windows 端**：设置 → 蓝牙 → 添加设备
2. **Android 端**：设置 → 蓝牙 → 搜索并配对 Windows 设备
3. 配对成功后，打开 SoundBridge
4. 设置 → 连接方式 → 选择"蓝牙"
5. 选择已配对的设备进行连接

**注意事项：**
- 蓝牙带宽有限，建议使用"超低延迟"音频模式
- 蓝牙 4.0+ (BLE) 或蓝牙 2.1+ 均支持

## 3. 功能说明

### 3.1 音频模式切换

SoundBridge 提供三种音频模式，根据使用场景选择：

| 模式 | 码率 | 延迟 | 适用场景 |
|------|------|------|----------|
| **均衡模式** | 96 kbps | ~40ms | 日常使用，兼顾音质和延迟 |
| **高音质模式** | 128 kbps | ~50ms | 听音乐、看电影 |
| **超低延迟模式** | 64 kbps | ~30ms | 游戏、实时通话 |

**切换方式：**
- **Windows**：托盘菜单 → 音频模式 → 选择模式
- **Android**：主界面 → 设置 → 音频模式 → 选择模式

### 3.2 混音比例调节

混音比例控制来自两端设备的音频音量占比。

- **手机→电脑**：控制手机音频在电脑端的播放音量
- **电脑→手机**：控制电脑音频在手机端的播放音量

**调节方式：**
- **Windows**：托盘菜单中的电平指示器下方有滑块
- **Android**：主界面的音量滑块

**典型配置：**
- 游戏场景：手机→电脑 80%（游戏音效为主），电脑→手机 20%（语音通话）
- 音乐场景：两端各 50%

### 3.3 静音控制

可单独静音任一方向的音频传输：

- **Windows**：托盘菜单 → 点击对应方向的静音图标
- **Android**：主界面 → 点击麦克风/扬声器图标

静音后该方向停止音频传输，但连接保持。

### 3.4 设备管理

**已配对设备列表：**
- **Windows**：托盘菜单 → 设备管理
- **Android**：设置 → 设备管理

**设备信息：**
- 设备名称
- 设备类型（Windows/Android）
- 连接方式
- 上次连接时间
- 自动连接开关

**设备记忆：**
- 首次连接成功后，设备自动保存到记忆列表
- 下次启动时，如开启"自动连接"，将自动重连上次设备
- 设备数据存储位置：`%LocalAppData%\SoundBridge\devices.json`

**删除设备：**
- 在设备列表中长按（Android）或右键（Windows）设备
- 选择"删除设备"

## 4. 快捷键

Windows 端支持以下全局快捷键：

| 快捷键 | 功能 | 说明 |
|--------|------|------|
| `Ctrl+Alt+P` | 切换传输方向 | 在"手机→电脑"和"电脑→手机"之间切换 |
| `Ctrl+Alt+M` | 切换混音模式 | 循环切换：均衡 → 高音质 → 超低延迟 → 均衡 |
| `Ctrl+Alt+S` | 打开设置 | 快速打开设置窗口 |

**快捷键特性：**
- 全局生效，即使 SoundBridge 不在前台也能使用
- 切换时系统托盘会显示通知提示当前状态
- 快捷键冲突时可在设置中自定义

## 5. 常见问题

### 5.1 连接失败

**问题：设备发现不到**

- 检查两端是否在同一 WiFi 网络
- 检查路由器是否开启 AP 隔离（需关闭）
- 尝试手动输入 IP 地址连接
- 重启两端的 SoundBridge 应用

**问题：连接后立即断开**

- 检查防火墙设置，允许 SoundBridge 通过
- 检查是否有多 SoundBridge 实例运行
- 查看系统日志是否有错误信息

**问题：ADB 连接失败**

- 确认手机已开启 USB 调试
- 运行 `adb devices` 确认设备已识别
- 重新插拔 USB 线
- 尝试更换 USB 端口或数据线

**问题：蓝牙连接失败**

- 确认设备已配对
- 重新配对设备
- 检查蓝牙驱动是否最新

### 5.2 音频卡顿

**问题：音频断断续续**

- 检查 WiFi 信号强度，靠近路由器
- 切换到 5GHz 频段（干扰更少）
- 关闭其他占用带宽的应用（如视频会议、下载）
- 尝试"超低延迟"模式（码率更低，抗丢包更强）

**问题：有杂音或爆音**

- 检查麦克风权限是否授予
- 降低混音比例，避免音量过大
- 尝试"均衡"模式

**问题：单向无声**

- 检查该方向是否被静音
- 检查系统音量设置
- 重启音频服务

### 5.3 延迟过高

**问题：音频延迟明显（>100ms）**

- 使用"超低延迟"模式
- 确保两端在同一 WiFi 网络（避免跨网段）
- 使用 5GHz WiFi 频段
- 尝试 USB/ADB 连接（延迟最稳定）
- 关闭路由器的 QoS/流量控制功能

**问题：延迟逐渐增大**

- 可能是网络拥堵，尝试切换信道
- 检查是否有设备在大量占用带宽
- 重启路由器

**问题：游戏场景延迟敏感**

- 推荐使用 USB/ADB 连接
- 使用"超低延迟"音频模式
- 关闭游戏内的音频后处理效果
- 降低游戏画质以减少 CPU 占用

### 5.4 其他问题

**问题：开机后 SoundBridge 没有自动启动**

- Windows 设置 → 应用 → 启动 → 确认 SoundBridge 已开启
- 或在 SoundBridge 设置中开启"开机自启动"

**问题：如何完全卸载 SoundBridge**

- **Windows**：控制面板 → 程序和功能 → 卸载 SoundBridge
- **Android**：长按应用图标 → 卸载
- 残留数据：`%LocalAppData%\SoundBridge\`（Windows）

**问题：如何查看日志**

- **Windows**：`%LocalAppData%\SoundBridge\logs\`
- **Android**：设置 → 关于 → 查看日志

## 附录

### A. 网络端口

| 用途 | 协议 | 端口 |
|------|------|------|
| 音频流 | UDP | 动态分配 |
| 控制信令 | UDP | 动态分配 |
| mDNS 发现 | UDP | 5353 |
| ADB 转发 | TCP | 5555 |

### B. 安全说明

- 所有音频数据通过 SRTP 加密传输（AES-128-CM）
- 密钥交换使用 ECDH (x25519) 协议
- 无数据上传到云端，所有传输在本地网络完成

### C. 技术参数

| 参数 | 值 |
|------|-----|
| 采样率 | 48000 Hz |
| 位深 | 32 bit (Float32) |
| 声道 | 单声道 |
| 帧大小 | 960 samples |
| 帧时长 | 20 ms |
