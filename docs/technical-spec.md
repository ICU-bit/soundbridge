## 1. 音频规格

### 1.1 采样参数
| 参数 | 均衡模式 | 高音质模式 | 超低延迟模式 |
|------|----------|------------|--------------|
| 采样率 | 48000 Hz | 48000 Hz | 48000 Hz |
| 位深 | 32 bit (Float32) | 32 bit (Float32) | 32 bit (Float32) |
| 声道 | 单声道 (1ch) | 单声道 (1ch) | 单声道 (1ch) |
| 帧大小 | 960 samples | 960 samples | 960 samples |
| 帧时长 | 20 ms | 20 ms | 20 ms |

### 1.2 Opus 编码参数
| 参数 | 均衡模式 | 高音质模式 | 超低延迟模式 |
|------|----------|------------|--------------|
| 码率 | 96 kbps | 128 kbps | 64 kbps |
| 复杂度 | 10 | 10 | 10 |
| 帧大小 | 20 ms | 20 ms | 20 ms |
| 带宽 | 全带宽 | 全带宽 | 全带宽 |
| 信号类型 | 自动 | 音乐 | 语音 |

### 1.3 延迟预算
```
总延迟 = 采集延迟 + 编码延迟 + 网络延迟 + 解码延迟 + 播放延迟

均衡模式：
- 采集延迟: 10-20 ms (WASAPI/AAudio 缓冲)
- 编码延迟: 20 ms (Opus 帧大小)
- 网络延迟: 5-20 ms (WiFi 局域网)
- 解码延迟: 5 ms
- 播放延迟: 10-20 ms
- 总计: 50-85 ms

超低延迟模式：
- 采集延迟: 5-10 ms
- 编码延迟: 10 ms
- 网络延迟: 5-15 ms
- 解码延迟: 2.5 ms
- 播放延迟: 5-10 ms
- 总计: 27.5-47.5 ms
```

## 2. 网络规格

### 2.1 UDP 音频流协议

#### 2.1.1 包格式
```
┌─────────────────────────────────────────────────────────────┐
│                     UDP 音频包格式                           │
├─────────────────────────────────────────────────────────────┤
│  字节 0-3:   序列号 (uint32_t, 网络字节序)                  │
│  字节 4-7:   时间戳 (uint32_t, 毫秒, 网络字节序)            │
│  字节 8:     标志位 (uint8_t)                               │
│              - bit 0: 是否关键帧                            │
│              - bit 1: 是否混音                              │
│              - bit 2-7: 保留                                │
│  字节 9:     通道数 (uint8_t)                               │
│  字节 10-11: Opus 数据长度 (uint16_t, 网络字节序)           │
│  字节 12-N:  Opus 编码数据                                  │
└─────────────────────────────────────────────────────────────┘
```

#### 2.1.2 丢包恢复
- 前向纠错 (FEC)：每个包包含前一帧的冗余信息
- 丢包隐藏 (PLC)：Opus 内置丢包隐藏
- 重传请求：关键帧丢失时请求重传

### 2.2 控制信令（UDP）

> **注意**：QUIC 控制信令尚未实现。当前控制信令通过 UDP 传输。

#### 2.2.1 消息格式
```
┌─────────────────────────────────────────────────────────────┐
│                     控制消息格式                             │
├─────────────────────────────────────────────────────────────┤
│  字节 0-3:   消息长度 (uint32_t, 网络字节序)                │
│  字节 4:     消息类型 (uint8_t)                             │
│  字节 5-N:   消息体 (JSON 或二进制)                         │
└─────────────────────────────────────────────────────────────┘
```

#### 2.2.2 消息类型
| 类型 | 代码 | 说明 |
|------|------|------|
| HELLO | 0x01 | 握手请求 |
| HELLO_ACK | 0x21 | 握手响应 |
| AUTH | 0x02 | 认证请求 |
| AUTH_ACK | 0x22 | 认证响应 |
| START_AUDIO | 0x03 | 开始音频传输 |
| STOP_AUDIO | 0x04 | 停止音频传输 |
| CHANGE_MODE | 0x05 | 切换音频模式 |
| VOLUME | 0x06 | 音量控制 |
| STATUS | 0x07 | 状态查询 |
| STATUS_ACK | 0x27 | 状态响应 |
| HEARTBEAT | 0x08 | 心跳包 |
| HEARTBEAT_ACK | 0x28 | 心跳响应 |
| ERROR | 0xFF | 错误消息 |

### 2.3 设备发现 (mDNS)

#### 2.3.1 服务类型
```
_soundbridge._udp.local.
```

#### 2.3.2 服务属性
| 属性 | 说明 | 示例 |
|------|------|------|
| id | 设备唯一ID | "device_001" |
| name | 设备名称 | "Pixel 7" |
| type | 设备类型 | "android" / "windows" |
| version | 协议版本 | "1.0" |
| capabilities | 支持的功能 | "audio,control" |

## 3. 音频处理规格

### 3.1 回声消除 (AEC)
- **算法**：WebRTC AEC3
- **尾部长度**：50 ms
- **延迟估计**：自动
- **采样率**：16000 Hz (内部处理)

### 3.2 噪声抑制 (NS)
- **算法**：WebRTC NS
- **模式**：中等抑制
- **采样率**：16000 Hz (内部处理)

### 3.3 自动增益控制 (AGC)
- **算法**：WebRTC AGC
- **目标电平**：-3 dBFS
- **增益范围**：0-30 dB
- **压缩比**：3:1

### 3.4 电平检测
- **窗口大小**：10 ms
- **更新频率**：30 Hz
- **显示范围**：-60 dBFS 到 0 dBFS
- **衰减时间**：300 ms

## 4. 连接管理规格

### 4.1 WiFi 局域网连接
```
设备发现流程：
1. 客户端广播 mDNS 查询
2. 服务端响应 mDNS 通告
3. 客户端解析服务属性
4. 建立 UDP 控制连接
5. 握手认证
6. 开始 UDP 音频流
```

### 4.2 WiFi 直连 (热点)
```
热点连接流程：
1. 用户在手机端开启热点
2. 电脑连接热点
3. 手机端启动服务
4. 电脑端手动输入 IP 或自动发现
5. 建立连接
```

### 4.3 USB 连接
```
USB 连接流程：
1. 手机开启 USB 调试
2. 电脑通过 ADB 连接
3. 端口转发 (adb forward)
4. 通过本地端口建立连接
```

### 4.4 蓝牙连接
```
蓝牙连接流程：
1. 设备配对
2. 建立 RFCOMM/SPP 通道
3. 或使用 BLE GATT 服务
4. 通过蓝牙通道传输控制信令
5. 音频流通过蓝牙音频协议 (A2DP/HFP) 或自定义通道
```

## 5. 安全规格

> **注意**：以下安全功能均为设计规格，当前版本（v0.7.2）**尚未实现**。音频和控制数据通过 UDP 明文传输。

### 5.1 传输加密（未实现）
- **控制信令**：UDP（当前实现），未来计划 TLS 加密
- **音频流**：可选 DTLS-SRTP（未实现）
- **密钥交换**：基于设备配对的预共享密钥（未实现）

### 5.2 设备认证（未实现）
- **配对方式**：
  - PIN码配对（未实现）
  - 二维码扫描配对（未实现）
  - 自动信任（同一网络）（当前行为：无认证）
- **会话令牌**：连接后生成会话令牌（未实现）

### 5.3 权限控制
- **Windows**：管理员权限（安装服务）、普通用户（日常使用）
- **Android**：录音权限、网络权限、前台服务权限
